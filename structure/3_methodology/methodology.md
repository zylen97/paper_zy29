## 必备元素

> **标题层级规则**：`###` = manuscript `\subsection`，`####` = manuscript `\subsubsection`。
> 下方列出本章需要的 subsection 及其颗粒度要求。初始化后根据实际研究替换 TODO 标题。

1. **3.1 {Methodology}**: (1) 方法简述（用了什么博弈/模型类型，主体结构，决策变量概览）；(2) 方法选择论证——基于 RQ 解释为什么此方法能回答研究问题、其他方法为什么不行（1-2 段文字论证即可，无需对比表）
2. **3.2 {Problem description and assumptions}**: 供应链结构 + 假设（每条附合理性论证） + 符号表（→ Table in tables.tex）
3. **3.3 {Model A / Benchmark model}**: 收益函数 + 均衡求解结果 + 经济直觉
4. **3.4 {Model B / Proposed model}**: 收益函数 + 均衡求解结果 + 与 Model A 的结构对比

> **颗粒度标准**（验证原则：sci-writer 翻译时不需要做任何技术判断）：
> 每个 `###` subsection 下必须包含：
> - 每一个会出现在正文中的公式（编号公式和关键行内公式）
> - 每一个命题/引理的完整陈述
> - 每一个证明的思路骨架（完整证明用 `> **Proof (→ Supplementary):**` 标记，sci-writer 外放至附录）
> - 每一个假设的文字表述 + 合理性论证
> - 每一个经济直觉/含义解读
> - 每一个模型间的对比说明
> - 表格和图表仅标注"此处引用 Table X / Fig. X"，内容在 `figures_tables/tables.tex`
